## English

### What it is

**Notes** is a Markdown note surface for session- and project-scoped notes. Files and attachments live under the shared panel home and are the **same files** the VS Code extension uses.

Desktop adds a fuller list / editor / preview experience plus optional indexing for search.

### Main workflows

1. Open the **Notes** tab.  
2. Create, open, rename, or delete notes.  
3. Edit Markdown; paste images as attachments when supported.  
4. Import existing Markdown into the notes system.  
5. **Reveal** a note in Finder, **copy path**, or **open the notes folder**.  
6. Search notes when indexing is available.

### Attachments & rename

- Images are stored as assets next to note content.  
- Use in-app rename so asset folders and relative links stay consistent.

### Shared with the extension

- Same on-disk Markdown + assets  
- Catalog index for presence / ownership  
- Extension optimizes for IDE capture; Desktop for longer editing  

See also [panel-doc Notes](https://github.com/lucacicii/agent-resume-panel-doc/blob/main/notes.md).

### Tips

1. Prefer project notes for durable docs; session notes for run-specific context.  
2. Back up panel home when migrating machines — [Settings & data](settings-and-data.md).  
3. Markdown preview is sanitized; treat notes as content, not trusted HTML.

## 简体中文

### 是什么

**Notes** 用于会话级 / 项目级 Markdown 笔记。文件与附件位于共用 panel home，与 VS Code 扩展 **同一批文件**。

Desktop 提供更完整的列表 / 编辑 / 预览，以及可选的索引搜索。

### 主要流程

1. 打开 **Notes** 页签。  
2. 新建、打开、重命名或删除笔记。  
3. 编辑 Markdown；支持时粘贴图片附件。  
4. 导入已有 Markdown。  
5. **在访达中显示**、**复制路径** 或 **打开笔记文件夹**。  
6. 在索引可用时搜索笔记。

### 附件与重命名

- 图片作为资源与正文一并管理。  
- 请用应用内重命名，以保持资源目录与相对链接一致。

### 与扩展共用

- 同一磁盘上的 Markdown 与资源  
- 目录索引负责归属与是否存在  
- 扩展侧重 IDE 快速记录；Desktop 侧重长文编辑  

另见 [panel-doc Notes](https://github.com/lucacicii/agent-resume-panel-doc/blob/main/notes.md)。

### 提示

1. 长期文档放项目笔记；单次运行上下文放会话笔记。  
2. 换机备份 panel home — [设置与数据](settings-and-data.md)。  
3. 预览会做安全处理；笔记按普通内容对待，不要当可信 HTML。
